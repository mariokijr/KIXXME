I want to make two major improvements to KixxMe.

1. KixxMe Live Gold (future feature)

I want KixxMe Live to become one of the main reasons users subscribe to Gold.

Concept:
A video-call section fully integrated inside KixxMe, not an external app.

Requirements:

* New tab with a camera icon: 📹 Live
* Only Gold members can access it.
* Only Gold members can be matched with other Gold members.
* Free and Plus users see a Gold paywall.
* Users cannot manually choose who to video call.
* Matching must be random and exciting.

Matching flow:

1. User opens Live.
2. User chooses filters:
    * Age range
    * Nearby users
    * Same city
    * Spain
    * Europe
    * Worldwide
3. User taps:
    🔥 Buscar videollamada
4. Show a fun waiting screen while searching.
5. When a match is found, show a pre-call reveal screen.

Searching screen:
Title:
“Buscando una conexión Gold para ti…”

Subtitle:
“Estamos encontrando a alguien con quien romper el hielo.”

Use a playful animated icon:
🎲
✨
📹
💫

Pre-call reveal screen:
Show:

* Profile photo
* Name
* Age
* City

Example:

“Has conectado con Eduardo”
“28 años · Madrid”

or

“Te vas a conectar con Eduardo, 28, Madrid”

Show a countdown:

“La llamada empieza en 5… 4… 3…”

Buttons:

* ✅ Aceptar
* 🔄 Siguiente
* ❌ Cancelar

Behavior:

* Aceptar = wait for both users to accept.
* Siguiente = skip and search for another random Gold user.
* Cancelar = stop searching.
* If both users accept, start the call.
* If either user cancels, return to searching.

Private video calls:

* Add a camera button inside private chats.
* A user can invite another user to a video call.
* The call starts only if both users are Gold and both accept.

In-call controls:

* Camera on/off
* Microphone on/off
* End call
* Report user
* Block user

Safety:

* Report and Block before and during calls.
* Show:
    “Respeta siempre a la otra persona. Puedes cancelar, bloquear o reportar en cualquier momento.”

Important:
Do not implement expensive video infrastructure yet.
Prepare the UI, database structure, routes and architecture for future WebRTC / LiveKit integration.
